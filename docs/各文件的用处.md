本文件提供了“工具” (gongju) 项目中提供的各脚本文件用处简述。

## bin 目录

build.sh

　　对系统进行构建，默认使用 step 文件所设置的顺序进行构建，有多个参数可设置个性化的构建过程，生成的系统存放在 overlaydir 目录中。


strip_os.sh

　　在系统构建完成后可使用该命令进行去除调试信息以及对系统按要求进行拆分，拆分后的系统存放在 overlaydir_strip 目录中。


install_os_run.sh

　　对已经拆分好的系统加装对应的运行脚本。


release_info.sh

　　按照设置获取构建系统中的软件包信息并汇总成发布说明文件。


pack_os.sh

　　对已经完成脚本安装后的系统组进行压缩打包，可指定打包格式。


live_os.sh

　　收集已经完成打包的系统组压缩文件并存放到指定目录中，生成一个可以启动计算机的文件目录结构。


set_released.sh

　　可以设置指定的组为 released 状态，处于该状态的组产生的更新文件将会聚到 update 目录中。


branch_create.sh

　　用于对当前构建设置、脚本以及版本信息进行收集并存放到指定名称的目录中进行保存，并将当前的构建目标设置为生成的名称，构建过程将默认使用该目录中的文件进行系统的构建，被保存的内容将不会因为git更新而发生变化。


branch_patch.sh

　　如果对构建过程的软件包制作脚本和软件包版本进行修改，可以通过本命令创建与指定保存目录中的对应内容生成差异文件。


branch_diff.sh

　　对 branch_patch.sh 脚本生成的差异文件进行应用，可应用到 branch_create.sh 保存的目录中。



## tools 目录

get_all_package_url.sh

　　用于下载指定软件包的源码及构建过程所需的资源文件，通过 step 中的记录可以下载相同组中的软件包或全部软件包。


git_clone.sh

　　若软件包使用git仓库来提供源码，则该脚本对指定git仓库进行复制并打包成压缩文件，通常以其它脚本调用的方式使用，不单独使用。


git_test.sh

　　测试指定软件包git仓库的更新需求是否满足，如需更新则返回 1 ，无需更新则返回 0 。通常以其它脚本调用的方式使用，不单独使用。


enter_package_env.sh

　　给命令用于进入指定软件包构建过程的编译环境中，会自动设置各种编译指定软件包所需要的环境变量。


show_package_script.sh

　　显示指定软件包的构建脚本内容，通常以其它脚本调用的方式使用，不单独使用。


run_package_script.sh

　　运行指定软件包的构建脚本内容，通常以其它脚本调用的方式使用，不单独使用。


strip_step.sh

　　对构建好的系统目录进行去除调试信息的步骤，通常以其它脚本调用的方式使用，不单独使用。


split_step.sh

　　对构建好的系统目录根据设置进行拆分的步骤，通常以其它脚本调用的方式使用，不单独使用。


final_step.sh

　　对构建好的系统目录按照指定的步骤进行最终的处理，通常以其它脚本调用的方式使用，不单独使用。


pack_archive_dir.sh

　　对 overlaydir_strip 目录中处理好的系统目录进行压缩处理，可指定压缩方式和格式，通常以其它脚本调用的方式使用，不单独使用。


import_pkg.sh

　　用于导入第三方已经编译好的软件包，目前支持 tar.gz、tar.xz、tgz 、rpm、deb 格式文件的导入。


merge_pkg.sh

　　对使用 import_pkg.sh 脚本导入的第三方软件包合并到指定的系统目录中，在系统目录压缩之前进行合并将成为系统目录内容的一部份。


get_version.sh

　　一个简单的获取指定步骤中的软件包版本信息脚本工具，需指定步骤组和软件包名称。


kernel_pack.sh

　　将指定目录中的内核及内核模块文件打包成可用于启动使用的目录和文件结构。


## env 目录

function.sh

　　该脚本用来给构建系统的过程中提供一些必要和方便构建的函数命令。


arch.data

　　该文件存放了各种架构的表示名称以及对应的配置文件名，各种架构的配置文件存放在 data 目录中。


data 目录

　　该目录中存放了各种架构文件的配置信息，当构建过程中使用到了对应的架构时可从对应的配置文件中获取需要使用的设置内容。
